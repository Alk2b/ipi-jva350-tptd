# IPI JVA350 - Qualité Logicielle

## Présentation du projet

Ce projet est réalisé dans le cadre du cours **JVA350 - Qualité Logicielle** à l'IPI.
Il porte sur la gestion des **congés payés des salariés aides à domicile**, en appliquant les règles
métier définies par la convention collective du particulier employeur.

## Objectifs pédagogiques

- Mettre en place une **intégration continue** avec GitHub Actions et SonarCloud
- Écrire des **tests unitaires** (simples et paramétrés) avec JUnit 5
- Écrire des **tests avec Mockito** en isolation totale
- Écrire des **tests d'intégration** avec Spring Boot Test
- Écrire des **tests d'acceptation** avec Cucumber
- Améliorer la **maintenabilité** du code (logging, conventions, documentation)

## Stack technique

| Technologie | Rôle |
|---|---|
| Java 17 | Langage principal |
| Spring Boot 4 | Framework applicatif et tests d'intégration |
| JUnit 5 | Tests unitaires et paramétrés |
| Mockito | Tests en isolation (sans base de données) |
| Cucumber | Tests d'acceptation |
| JaCoCo | Couverture de code |
| SonarCloud | Analyse de qualité continue |
| H2 | Base de données en mémoire pour les tests |
| MkDocs Material | Cette documentation |
